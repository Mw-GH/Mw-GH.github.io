  7. 线程的几种状态

    NEW：初始状态
    线程还没有启动时的状态。

    RUNNABLE：就绪状态和执行状态
    线程启动时的状态。线程被JVM所执行但它还可能需要一些来自操作系统的其他资源才能执行。

    BLOCKED：阻塞状态 当线程访问临界区 (同步块代码)时，如果没有拿到访问锁，便进入阻塞状态。
    线程被一个监听锁所阻塞时的状态。

    WAITING：无限期等待状态，线程正在等待时的状态。线程被以下方法所调用就会进入等待状态：

    • Object.wait无参方法 wait方法可以让当前线程进入等待状态，需要其他线程调用此线程对象的notify方法或者notifyAll方法来唤醒此线程；
    • Thread.join无参方法 调用join方法的线程需要等到被调用线程终止才能结束等待状态。

    TIMED_WAITING： 有限等待状态
    线程在指定时间后才能结束等待的一种等待状态。是由于调用了以下方法所引起的一种状态：

    • Thread.sleep
    • Object.wait带参方法
    • Thread.join带参方法

    TERMINATED ：终止状态
    线程终止时的状态。该线程已经执行完毕。

  5. 数据库B+树索引

    mysql中的索引是以文件的形式按B+树数据结构存放在磁盘当中，索引指向着对应的数据

    B+树是B树演进的，B树又是二叉树演进的

    二叉树的特性：一个节点最多只能有二个子节点，除开极端条件，当数据量变大时二叉树的高度就会随之变高，磁盘IO次数随之变多，效率低

    B树特性：一个节点存放多个元素 （m/2 <= k <= m），数据量变大时B树的，比起二叉树 高度增加变低，磁盘IO次减少，效率变高

    B+树：一般只要3层 就能满足千万级别的数据，因为非叶节点仅存放键值，叶节点存放索引，所以高度通常不会增加，3次磁盘IO，查询效率高，且稳定，

    一个节点（页）大小为16K

    一个非叶节点 16K/14=1170个指针 （14=主键 ID 为 bigint 类型，长度为 8 字节 + 指针大小在 InnoDB 源码中设置为 6 字节）

    一个叶节点，一行大约为1K 能记录16K/1K=16条

    一个指针指向一个叶节点也就是指向16条记录

    一棵高度为2的B+树 能存放：1170*16=18720条记录 （索引）

    一棵高度为3的B+树 能存放：1170*1170*16=21902400 条记录 2千万

    5.1. 为什么 MySQL 的索引要使用 B+ 树而不是其它树形结构？

    比如 B 树？

    因为 B 树不管叶子节点还是非叶子节点，都会保存数据，这样导致在非叶子节点中能保存的指针数量变少，指针少的情况下要保存大量数据，只能增加树的高度，导致 IO 操作变多，查询性能变低。

  9. 多线程start和run方法。

    答：run()方法只是一个类的普通方法，调用后自动压栈执行，执行完后弹栈便可

    ​ start()开辟空间——分支线程调用run()共同抢夺CPU资源抢占时间片执行代码——start()执行结束
