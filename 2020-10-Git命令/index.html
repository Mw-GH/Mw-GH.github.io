Git命令

常用命令

  • git clone https://github.com/Mw-GH/GitLearning.git 从远程仓库上克隆项目

  • git push -u orgin master
    将本地的最新数据推送到远程仓库
    第一加 -u后 后面可直接使用 git push
    pull同理

  • git pull orgin master
    将远程仓库的最新数据同步到本地

    准备

  • git config user.name
    查看配置用户

  • git config user.email
    查看配置邮箱

  • git config --global user.name "Mw-GH"
    配置更改git用户

  • git config --global user.email "Mw-MS@outlook.com"
    配置或更改git邮箱

  • ssh-keygen -t rsa -C "Mw-MS@outlook.com"
    在C:\Users\Administrator目录下生成.ssh

版本管理

开始

  • git init
    初始化仓库

  • git add <file>
    更新文件 工作区>>>暂存区

  • git commit -m <"describe">
    更新版本 暂存区>>>仓库（版本库）

    注意
    一次commit能够提交上一次commit后所有的修改添加

  • git checkout <file>
    git checkout . 恢复全部文件的修改
    恢复修改 撤销
    未git add的文件

  • git reset HEAD
    撤销文件更新 工作区<<<暂存区
    已经git add的文件，再用上一条命令恢复文件
    同commit 全部恢复为未提交状态

  • git reset --hard HEAD^
    git rest --hard <版本号>
    版本回退 暂存区<<<仓库（仓库）
    回退到上一个版本 上上一个版本为HEAD^^ 上一百个版本HEAD~100
    已经commit后 恢复所有文件为上一个版本

  • git rm test.txt
    删除test.txt文件

  • git status
    查看commit前仓库的状态 是否有需要提交的文件

  • git diff [file]
    查看commit前的差异 difference差异

  • git log
    查看最近到最远的提交日志
    git log --pretty=oneline
    简略版日志

  • git reflog
    查看你的每一次命令

分支管理

branch

  • git checkout -b dev
    创建并切换到新的分支dev
    等同于
    git branch dev
    git checkout dev

  • git branch
    查看当前所有分支

  • git merge dev
    当前分支master合并dev分支

合并方式：
Fast-forward:快进模式git merge dev，直接把master指向dev的提交，缺点：看不出合并记录还会搅乱master分支提交记录
禁用Fast-forward:普通模式git merge --no-ff -m "merge --no-ff",合并后的历史有分支，能看出来曾经做过合并

git merge --no-ff -m "merge with no-ff" dev
禁用Fast forward模式 合并分支并生成一个commit

建议使用
--no-ff参数 就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

  • git branch -d dev
    删除dev分支

switch

  • git switch -c dev
    创建并切换到新的分支dev

  • git switch master
    直接切换到已有分支master

注意
因为创建、合并和删除分支非常快，所以Git鼓励你使用分支完成某个任务，合并后再删掉分支，这和直接在master分支上工作效果是一样的，但过程更安全。

  • git log --graph --pretty=oneline --abbrev-commit
    用带参的git log查看分支的合并情况

stash

  • git stash
    git stash save "sta1"
    储藏现在的工作区

  • git stash list
    查看所有储藏的内容

  • git stash pop
    恢复内容同时删除stash

  • git stash apply
    git stash apply stash@{0}
    恢复最近一个stash

  • git stash drop stash@{0}
    删除指定stash

  • git stash branch
    从最新的stash创建分支

  • git cherry-pick
    复制一个特定的提交到当前分支
